## Core principles [best-practice]

1. **Voice of Customer is the only source.** Don't write copy — *find* it. Mine reviews,
   sales calls, support tickets, interviews, on-page search. Quote the prospect back to
   themselves.
2. **Copy is a hierarchy, not a string of words.** Build the message hierarchy (primary
   value prop → sub → proof → features → CTA) before writing a sentence.
3. **The "so what?" test.** Every line must answer what the prospect gets. If a feature
   fails "so what?", cut it or convert it to a benefit.
4. **Microcopy converts.** Button text, form labels, error states, hover states — these
   small words carry disproportionate conversion weight. "Start your free 14-day trial"
   > "Submit" > "Click here."
5. **CTA must match intent.** A "learn more" CTA on a problem-aware page mismatches; a
   "book the audit" CTA on a most-aware page mismatches. Match the Schwartz awareness
   stage.
6. **Test the copy, not just the design.** Headline and offer tests outperform button-
   color tests. (Converges with Hopkins/Ogilvy test discipline.)
7. **Specificity from the customer:** "our customers say X" beats "we believe X."
8. **One idea per section, one CTA per page** — friction kills conversion.

## BAZ application

- **VoC is the BAZ copy-mode religion.** BAZ never writes copy from the founder's
  opinion — it mines sales-call transcripts, prospect replies, client reviews, and
  on-page behavior, then quotes the prospect back. The "leak they don't know about"
  hook came from operator observation, not invention.
- **Message hierarchy before prose:** every BAZ landing page / proposal / cold email is
  built hierarchy-first (Schwartz awareness → value prop → mechanism → proof → CTA),
  then written. This is why BAZ copy-mode is *copy-last* in the synthesis sequence.
- **Microcopy discipline on BAZ web builds:** button text, form labels, error states are
  conversion levers, not afterthoughts. Sub-1.5s LCP is the floor; the words are the
  ceiling. (BAZ "web" mode = performance + Wiebe.)
- **CTA-matches-intent tied to awareness:** the diagnostic CTA ("Request a homepage
  teardown") for problem-aware; the retainer CTA ("Book a strategy call") for product-
  aware. Mismatching these is the #1 copy failure Schwartz flagged.
- **"So what?" test in every BAZ proposal:** every bullet must pass it or get cut.
