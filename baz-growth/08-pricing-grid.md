# 08 — Pricing grid (the three tiers, calibrated)

> The full pricing surface. Project + retainer, never SaaS. Calibrated to
> buyer stage. The senior-partner rule: recommend the tier that fits the
> buyer's stage; don't oversell Growth to a Core-fit client.

## The grid

| Tier | Price | Term | Scope | Who it's for | Delivery |
|---|---|---|---|---|---|
| **Project** | $12k–$80k fixed | 4–14 weeks, scoped finish line | one defined deliverable (tracking setup, web rebuild, SEO sprint, audit) | the buyer who needs proof before a retainer, or a one-off | senior partner, fixed scope |
| **Core** | $6,500–$9,500/mo | 90-day min, then monthly | one channel (paid) + tracking + weekly demo | post-product-fit brand, $20k–$500k/mo media spend | one senior partner |
| **Growth** | $18k–$28k/mo | quarterly | 3–5 senior partners, full integrated stack (paid + SEO + content + influencer + web + data) | the brand scaling across channels, $500k+/mo spend | partner team |

## Band detail — Core

| Client media spend (monthly) | Core price | Why |
|---|---|---|
| $20k–$100k | $6,500/mo | one channel, lean management |
| $100k–$300k | $8,000/mo | more creative + iteration |
| $300k–$500k | $9,500/mo | full senior-partner load |

> > $500k/mo → Growth tier. Do not extend Core.

## Band detail — Project

| Project type | Price range | Weeks |
|---|---|---|
| Tracking + attribution setup (server-side GTM/GA4/CAPI) | $12k–$22k | 4–6 |
| Performance web rebuild (Next.js + Supabase, sub-1.5s LCP) | $25k–$60k | 8–14 |
| SEO/AEO sprint (24-type content surfaces + technical SEO) | $15k–$35k | 6–10 |
| Growth audit + 90-day roadmap | $12k–$18k | 4 |

> Project is the **entry/proof door**. A satisfied project client is the
> easiest Core/Growth upgrade. Always quote the upgrade path at project close.

## Band detail — Growth

> Growth is the **flagship**. It's where BAZ's integrated senior-team model

> One pricing structure, three message calibrations. Never discount by
> region — discount destroys the senior-partner positioning. Calibrate the
> message, not the price.

## The tier-selection rule (internal)

| Buyer signal | Quote |
|---|---|
| Pre-product-fit, < $20k/mo spend | Project (audit) or pass |
| Post-fit, one channel pain, $20k–$500k spend | Core |
| Post-fit, multi-channel pain, $500k+ spend | Growth |
| Wants a one-off (tracking, web, audit) | Project, with upgrade path quoted |
| Asks for SaaS / self-serve | politely decline; BAZ is not SaaS |

## Guardrails (non-negotiable)

- **Never SaaS subscriptions.** Retainer or fixed project, always.
- **Never discount to win.** Discount = junior-pod positioning. Calibrate scope down (Core) before cutting price.
- **Never oversell Growth to a Core-fit buyer.** The senior-partner model means the right tier, not the biggest tier.
- **Never quote a price before the audit reveals the gap.** Price lands after value is visible.
- **Never hide the exit criterion.** It's in the SOW for every retainer.

## Flags

- All prices are the canonical BAZ pricing — verify against the current
  brand standard before quoting. If a proposal uses a different number,
  this grid wins.
- Project pricing assumes the deliverable is scoped with an exit line. No
  open-ended project — that's a retainer dressed as a project.
